# 交互案例知识库：最小场景（Demo）

## 场景概述

- 空域：矩形区域 0..100km x 0..100km
- 平台：3 架无人机（UAV_1/2/3）+ 地面站（GS）
- 角色：
  - GS：全局决策与任务调整
  - UAV_1：簇首（中层决策、融合与重分配）
  - UAV_2/UAV_3：成员（执行与局部自主）

## 关键约束

- 总带宽受限：上行/下行总带宽 < 1Mbps
- 链路随机丢包：允许短时间失联

## 典型时序（简化）

1. 初始化：GS 将区域按条带划分给 UAV_1/2/3（可用 CMD_MISSION 广播）
2. 搜索：UAV_* 每 5s 上报 UAV_STATUS 给 GS
3. 发现目标：
   - UAV_2 或 UAV_3 发现目标，向簇首发送 TARGET_REPORT
   - 簇首融合并形成 TARGET_SUMMARY 上报 GS
4. GS 响应：
   - 若目标威胁等级高，GS 在 5s 内下发 CMD_ACTION（跟踪/观察/忽略）
   - 簇首收到后下发 CLUSTER_CMD 指派成员短时跟踪
5. 降级策略：
   - 若 GS 在一定超时内无响应，簇首可在权限范围内临时重分配，但必须保证至少 1 架机持续搜索覆盖
